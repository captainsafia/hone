## Key Design Decisions

1. **SwiftUI + AppKit Hybrid**
   - SwiftUI for modern, declarative UI
   - AppKit for system integration (menu bar, hotkeys)

2. **No In-App Editor**
   - Opens GitHub for final editing
   - Reduces complexity
   - Leverages GitHub's mature UI

3. **Multi-LLM Support**
   - User choice and flexibility
   - No vendor lock-in
   - Automatic selection based on available keys

4. **Private Gists for Attachments**
   - Native GitHub integration
   - No separate storage needed
   - Private by default

5. **Device Flow OAuth**
   - No client secret needed
   - Safe for native apps
   - Standard GitHub practice

6. **Graceful Degradation**
   - Works without LLM keys
   - Fallback formatting always available
   - Never blocks user

7. **Keychain Storage**
   - macOS system security
   - Encrypted at rest
   - No plain text secrets

## Performance Considerations

- **Async/Await**: All network calls non-blocking
- **Main Actor**: UI updates on main thread
- **Lazy Loading**: Services initialized on demand
- **Memory**: Minimal footprint (menu bar app)
- **Startup**: Fast initialization
- **Background**: No unnecessary background work
